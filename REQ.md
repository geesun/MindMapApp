# 桌面端思维导图应用（MindMap）需求规格说明书

## 1. 项目概述

### 1.1 术语定义

| 术语         | 定义                                                         |
| ------------ | ------------------------------------------------------------ |
| 根节点       | 思维导图的中心主题（仅 1 个），对应 Markdown 的 # 标题        |
| 一级子节点   | 根节点下的第一层节点，对应 Markdown 的 ## 标题               |
| 同级节点     | 同一层级的节点（如两个一级子节点互为兄弟节点）               |
| 子节点       | 某节点下的下一级节点（如一级子节点下的 ### 标题节点）        |
| .mind 格式   | 应用自定义的思维导图存储格式（JSON 结构，包含节点信息、样式、布局） |

### 1.2 排除范围

- 暂不支持多人协作、云端同步功能
- 暂不支持思维导图模板库、主题库
- 暂不支持导入/导出 Excel、Word 等非思维导图/Markdown 格式
- 暂不支持节点添加备注、标签、附件等扩展属性
### 1.3 项目目标

开发一款轻量级、跨平台的桌面端思维导图应用，支持通过导入 Markdown 文件自动生成层级化思维导图，并提供可视化的编辑、样式定制、导出功能，满足用户快速创建、编辑、分享思维导图的需求。

### 1.4 核心定位

- **目标用户**：学生、职场人士、创作者（需要快速将文本转化为思维导图的人群）
- **核心优势**：Markdown 一键导入生成思维导图、轻量化、操作简单、支持本地文件管理
- **对标产品**：XMind（核心功能对齐，简化非核心复杂功能）

## 2. 核心功能需求

### 2.1 基础文件管理

| 功能点         | 详细需求                                                                 |
| -------------- | ------------------------------------------------------------------------ |
| 新建思维导图   | 支持创建空白思维导图，默认生成根节点（中心主题，文本默认“未命名主题”）；支持创建时选择初始布局（如放射状/树形） |
| 打开本地文件   | 1. 支持打开本地的 .mind（自定义格式）.md 文件；<br>2. 打开.md 文件时自动触发“Markdown 导入生成思维导图”流程；<br>3. 打开文件时校验格式合法性，非法格式弹窗提示“文件格式错误，无法打开” |
| 保存文件       | 1. 手动保存：点击保存按钮/快捷键 Ctrl+S，直接覆盖原文件；<br> |
| 另存为         | 1. 支持将文件另存为 .mind、.md、；<br>2. 另存为.md 时，按节点层级还原为对应标题/列表格式（保留加粗/斜体）；<br> |
| 关闭文件       | 1. 关闭当前编辑的思维导图，未保存时弹窗提示“是否保存当前修改？”（是/否/取消）；<br>2. 关闭应用时，若有未保存文件，批量提示保存 |

### 2.2 Markdown 导入生成思维导图（核心功能）

| 功能点             | 详细需求                                                                                       |
| ------------------ | -------------------------------------------------------------------------------------------- |
| Markdown 解析规则  | 1. 标题层级映射：# → 根节点（仅第一个#标题为根节点，后续#标题降级为##），## → 一级子节点，### → 二级子节点（最多支持 8 级层级）；<br>2. 列表映射：-/*/1. 开头的列表项映射为对应层级的子节点（如## 标题\n- 列表项 → 标题节点下的子节点）；<br>3. 加粗/斜体：**文本**/*文本* 保留为节点文本的样式（加粗/斜体）；<br>4. 忽略规则：忽略图片（![]()）、链接（[xxx](xxx)）、代码块（```）、表格、换行符等非文本元素；<br>5. 空行处理：Markdown 中的空行不影响节点层级，自动忽略 |
| 导入流程           | 1. 点击“导入 Markdown”→弹窗选择本地.md 文件→预览 Markdown 文本（支持滚动）→点击“确认导入”；<br>2. 解析完成后自动生成可视化思维导图，默认布局为“放射状”；<br>3. 解析失败场景：<br>- 文件不存在/权限不足：提示“文件无法访问，请检查路径或权限”；<br>- 无有效标题/列表：提示“Markdown 文件无有效内容，无法生成思维导图”；<br>- 格式异常（如层级错乱）：提示“Markdown 格式异常，已尝试最大程度解析”并生成部分节点 |
| 预览功能           | 1. 导入前：弹窗预览 Markdown 文本（只读，支持语法高亮）；<br>2. 导入后：支持在侧边栏切换“思维导图视图/Markdown 原文视图”对比查看 |

### 2.3 思维导图可视化编辑

| 功能点     | 详细需求                                                                                       |
| ---------- | ---------------------------------------------------------------------------------------------- |
| 节点操作   | 1. 添加节点：<br>- 右键菜单：选中节点→右键→“添加子节点”/“添加同级节点”；<br>- 快捷键：Tab（创建子节点）、Enter（创建同级节点）、Shift+Tab（升级为父节点的同级节点）；<br>2. 删除节点：<br>- 选中节点后右键/Delete 键删除，删除父节点时弹窗提示“是否删除该节点及所有子节点？”（是/否）；<br>3. 编辑节点文本：<br>- 双击节点进入编辑模式（输入框），支持富文本（加粗 Ctrl+B/斜体 Ctrl+I），回车确认编辑，ESC 取消；<br>4. 移动节点：<br>- 拖拽节点调整层级（如将二级子节点拖到根节点下变为一级子节点），拖拽时显示层级辅助线；<br>- 拖拽节点调整位置（如调整同一层级内节点的显示顺序）；<br>5. 复制/粘贴节点：<br>- 快捷键 Ctrl+C/Ctrl+V，支持单个/框选多个节点复制，粘贴后保留原层级/样式；<br>- 剪切：Ctrl+X，剪切后节点暂存，粘贴后原节点删除 |
| 样式定制   | 详见下方“样式定制详细说明” |
| 视图操作   | 详见下方“视图操作详细说明” |
#### 样式定制详细说明

- **节点样式**：
	- 基础样式：背景色（16 进制色值选择/预设色板）、文字颜色、字体（系统字体列表）、字号（12-24px）、边框（有无/圆角/粗细）
	- 样式应用：支持“仅当前节点”/“当前节点及所有子节点”/“所有节点”三种应用范围
- **连线样式**：
	- 基础样式：颜色、粗细（1-8px）、线型（实线/虚线/点线）
	- 样式应用：全局生效（所有连线）
- **布局切换**：
	- 支持布局类型：放射状（中心向外）、树形布局（左到右/上到下）、鱼骨图布局
	- 切换后实时刷新画布，节点位置自动适配

#### 视图操作详细说明

- **缩放**：
	- 鼠标滚轮缩放（缩放范围：50%-200%）
	- 快捷键：Ctrl++（放大）/Ctrl+-（缩小）、Ctrl+0（恢复 100%）
- **平移**：拖拽画布空白处移动视图
- **居中**：点击“居中”按钮，思维导图自动居中显示
- **全屏**：F11 快捷键/点击“全屏”按钮，全屏模式下隐藏菜单栏外的无关元素，ESC 退出全屏

### 2.4 导出功能

| 功能点     | 详细需求                                                                                       |
| ---------- | ---------------------------------------------------------------------------------------------- |
| 导出格式   | 1. 图片格式：PNG（支持透明背景）、SVG（矢量图，无损缩放）；<br>2. 分辨率配置：<br>- PNG：默认 1920×1080，支持自定义（最大 4K：3840×2160）；<br>- SVG：无分辨率限制（矢量）； |
| 导出配置   | 1. 导出图片时可选择：<br>- 显示范围：“完整思维导图”/“选中节点及子节点”；<br>- 背景设置：“透明背景”/“白色背景”/“自定义背景色”；<br>- 画质：“普通”/“高清”（PNG 专属）；<br>2. 导出路径：默认上次导出路径，支持自定义，导出后弹窗提示“导出成功，是否打开文件夹？” |

### 2.5 快捷键支持（提升操作效率）

| 快捷键      | 对应功能           |
| ----------- | ------------------ |
| Ctrl+N      | 新建思维导图       |
| Ctrl+O      | 打开本地文件       |
| Ctrl+S      | 保存文件           |
| Ctrl+Shift+S| 另存为             |
| Ctrl+Z      | 撤销操作（最多撤销 20 步） |
| Ctrl+Y      | 恢复操作（最多恢复 20 步） |
| Tab         | 选中节点后，创建子节点 |
| Enter       | 选中节点后，创建同级节点（兄弟节点） |
| Shift+Tab   | 选中节点后，升级为父节点的同级节点 |
| Delete      | 删除选中节点       |
| Ctrl+B      | 选中节点文本后，设置/取消加粗 |

| Ctrl+0      | 恢复画布缩放为 100% |

## 3. 非功能需求

### 3.1 性能需求

- 解析 Markdown 文件：≤1000 行的 Markdown 文件解析时间≤1 秒；≤5000 行的解析时间≤3 秒；
- 响应速度：节点操作（添加/删除/移动）响应时间≤0.5 秒；布局切换响应时间≤1 秒；
- 内存占用：运行时内存占用≤200MB（普通思维导图，节点数≤1000）；节点数≤5000 时，内存占用≤500MB；
- 启动速度：应用冷启动时间≤3 秒（Windows/macOS）。

### 3.2 兼容性需求

- 操作系统：
	- Windows：Windows 10/11（64 位），兼容主流版本（21H2 及以上）；
	- macOS：macOS 10.15+（Catalina 及以上）；
	- Linux：Ubuntu 20.04+/CentOS 8+（64 位）；
- 文件兼容：
	- .md 文件：兼容 CommonMark 标准的 Markdown 文件，不兼容 GFM（GitHub Flavored Markdown）的表格、任务列表等扩展语法；
	- .xmind 文件：兼容 XMind 8 及以上版本的核心结构（节点层级、文本），不兼容样式、附件等扩展内容；
- 字体兼容：仅支持系统内置字体，不支持自定义字体文件导入。

### 3.3 易用性需求

- 界面风格：
	- 遵循各操作系统的原生设计规范（Windows 用 Win11 原生控件，macOS 用 Cocoa 风格）；
- 首次启动时显示“新手引导弹窗”（3 步：导入 Markdown→编辑节点→导出图片），支持“不再显示”；
- 错误提示：
	- 提示样式：系统原生弹窗（带图标：警告/错误/信息）；
	- 提示内容：包含“问题+解决方案”（如“Markdown 格式错误，无法解析 → 请检查标题层级是否连续，或列表是否嵌套正确”）；
- 操作反馈：节点点击/拖拽/编辑时，给出视觉反馈（如选中节点高亮边框、拖拽时半透明预览）。

### 3.4 稳定性需求

- 崩溃率：常规操作下崩溃率≤0.1%；连续操作 1 小时无崩溃；
- 数据安全：
	- 自动保存文件保留最近 3 个版本，防止单版本损坏；
	- 应用异常退出后，重启时自动恢复最后一次自动保存的内容；
	- 保存/导出时，若路径不可写（如只读文件夹），自动切换到系统桌面并提示用户。

### 3.5 可维护性需求

- 代码规范：遵循主流规范（如 JS/TS 遵循 ESLint Airbnb 规则，Python 遵循 PEP8）；
- 日志记录：记录关键操作日志（文件打开/保存/导入/导出、崩溃信息），日志存储路径：系统默认日志目录，最大日志大小 100MB，自动清理超过 30 天的日志；
- 配置文件：应用配置（自动保存间隔、默认导出路径等）存储为 JSON 格式，用户可手动修改。

## 4. 技术约束

### 4.1 开发框架优先级

- 首选：Electron（v28+）+ TypeScript + React/Vue3（前端 UI）；
- 备选：Tauri（v1.5+）+ Rust + TypeScript；
- 禁止：使用闭源商业 SDK/库（避免版权问题）。

### 4.2 核心依赖库建议

| 功能模块     | 推荐库（Electron 方案）         | 版本要求         |
| ------------ | ------------------------------- | --------------- |
| Markdown 解析| marked.js                       | v12+，兼容 CommonMark |
| 思维导图渲染 | mind-elixir-vue / mxGraph        | v2+/v4.2+，支持拖拽/层级调整 |
| 文件操作     | fs-extra                        | v11+，兼容跨平台路径 |
| 导出图片/SVG | html2canvas + svg2pdf.js         | v1.4+/v2+，支持高清导出 |
| 配置管理     | electron-store                   | v8+，跨平台配置存储 |

## 5. 界面原型（文字描述，供 AI 参考）

### 5.1 主界面布局

```
┌─────────────────────────────────────────────────────────────┐
│ 菜单栏（文件/编辑/视图/帮助）│ 工具栏（图标+文字，支持悬停提示） │
├───────────────────────────────────────────┬─────────────────┤
│ 左侧画布（思维导图编辑区域）                  │ 右侧面板        │
│ - 画布背景：默认浅灰色（#F5F5F5）| - 样式设置（折叠面板）│
│ - 辅助线：显示节点层级辅助线（可关闭）|   - 节点样式      │
│                                           │     □ 背景色选择器 │
│                                           │     □ 文字颜色选择器 │
│                                           │     □ 字体/字号下拉框 │
│                                           │   - 连线样式      │
│                                           │ - 布局设置（折叠面板）│
│                                           │   □ 放射状 □ 树形（左→右） □ 鱼骨图 │
│                                           │ - 操作日志（折叠，可选）│
└───────────────────────────────────────────┴─────────────────┘
```

### 5.2 关键弹窗原型

| 弹窗名称             | 触发场景                         | 内容/操作按钮                                                                 |
| -------------------- | -------------------------------- | ----------------------------------------------------------------------------- |
| 导入 Markdown 预览弹窗 | 点击“导入 Markdown”并选择文件后   | 标题：“Markdown 预览”<br>内容：Markdown 文本预览框（只读，语法高亮）<br>按钮：“确认导入”/“取消” |
| 删除节点确认弹窗     | 删除含子节点的父节点时           | 标题：“确认删除”<br>内容：“删除该节点将同时删除其下所有子节点，是否继续？”<br>按钮：“是”/“否” |
| 保存提示弹窗         | 关闭未保存文件/应用时            | 标题：“未保存的修改”<br>内容：“当前思维导图有未保存的修改，是否保存？”<br>按钮：“保存”/“不保存”/“取消” |
| 导出配置弹窗         | 点击“导出”时                     | 标题：“导出设置”<br>内容：1. 格式选择：PNG/SVG（单选）<br>2. 分辨率（PNG 专属）：默认/自定义（输入框）<br>3. 背景设置：透明/白色/自定义（单选+色值选择）<br>按钮：“导出”/“取消” |
